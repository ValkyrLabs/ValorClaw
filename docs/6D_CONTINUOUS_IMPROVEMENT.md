# 6D Continuous Improvement

## Purpose

The 6D loop gives ValorClaw instances a practical rhythm for continuous improvement without requiring ceremonial overhead.

Use it during normal operations, reviews, heartbeats, and weekly resets.

## The 6 disciplines

### 1. Decide

Clarify what matters now.

Questions:
- What is the real objective?
- What is noise?
- What is blocked?
- What deserves the heavy lane?

### 2. Define goals

Convert intent into explicit outcomes.

Questions:
- What does done look like?
- What evidence proves success?
- What is the smallest shippable slice?

### 3. Design systems

Shape the environment so good execution becomes easier.

Examples:
- better runbooks,
- better defaults,
- stronger templates,
- safer automation,
- cleaner delegation,
- more reliable memory paths.

### 4. Do the work

Execute with discipline.

Rules:
- start the real work,
- avoid proof-of-work theater,
- validate with canonical tests,
- close loops quickly,
- clean as you go.

### 5. Digest learnings

Turn raw experience into durable improvements.

Capture:
- failures,
- surprises,
- bottlenecks,
- repeated confusion,
- better rules discovered during execution.

### 6. Defend and deepen

Protect the gains and improve resilience.

Examples:
- prevent recurrence,
- update templates,
- harden docs,
- improve cleanup routines,
- strengthen memory habits,
- make delegation more robust when specialists are missing.

## Where to encode the improvements

Use:
- GrayMatter for durable operational memory,
- AGENTS templates for behavioral rules,
- runbooks/docs for repeatable processes,
- operating dashboards and trackers for active execution.

## Weekly questions

- What produced real results?
- What kept repeating unnecessarily?
- What should become a hard rule?
- What should be deleted?
- What should be automated?
- What should be delegated differently?

## Anti-patterns

Avoid:
- endless planning,
- non-durable insights,
- noisy “self-improvement” with no behavioral effect,
- scripts or helpers that never become real operating paths,
- letting cleanup and resilience work be deferred forever.
